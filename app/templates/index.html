1.1 · TWO-VIEW EVOLUTION

召回与排序双视角的完整技术演进

推荐算法的发展不只是“模型越来越深”。每一轮变化都在重新权衡表示能力、监督信号、全库检索成本与在线延迟：召回要从海量目录里快速找对方向，排序要在有限候选里恢复更强交互。

1990s—2013

从邻域关系到低秩表示

GroupLens 与 ItemCF 从相似用户/物品借信号；MF 把稀疏交互压成 user/item 坐标；FM 再把低秩交互推广到任意稀疏特征。此时召回与排序尚未完全分工。

2013—2017

双塔检索与深度排序定型

DSSM 让 item 向量可预计算并接入 ANN；GBDT+LR、Wide&Deep 与 DeepFM 在较小候选集恢复复杂交互，现代级联漏斗由此成形。

2017—2023

从静态用户到兴趣状态

MIND 用多个向量覆盖并行兴趣，DIN/DIEN 让表示依赖候选与时间，SASRec 以因果自注意力建模行为序列，MMoE/PLE 则把多个业务目标的共享与冲突显式化。

2023—2026

Semantic ID 与生成统一

TIGER 把全库检索转成受约束 Semantic ID 生成；OpenOneRec 探索列表生成与偏好对齐；HSTU 把长期行为流视为统一序列。新瓶颈转向合法解码、目录更新、吞吐与 GPU ROI。

如何理解这条主线

级联系统并未“过时”，它仍是延迟、成本与可控性约束下的帕累托解。生成式推荐的价值，在于把更强交互逐步前移，并尝试用统一 token 与训练目标减少阶段错配；是否值得替换现有漏斗，最终仍要由端到端收益和服务成本共同决定。

时间轴表示技术重心变化，不表示旧方法被完全替代。关键原始来源可在 附录 A.1 论文清单 中逐篇查阅。
