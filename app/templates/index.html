01 · TWO-VIEW EVOLUTION

召回与排序双视角的完整技术演进

召回追求全库效率，排序追求候选内精度。两者的结构性张力，决定了双塔可预计算、交叉模型难索引，也解释了生成式推荐为何尝试重画边界。

核心判断

级联系统没有“过时”：它是延迟、成本、可控性约束下的帕累托解。生成式系统的进步，是逐步把更强交互前移，并用 token 化和统一目标降低阶段错配。

主线依据包括 ItemCF、DSSM、YouTube DNN、TIGER 与 HSTU。时间轴表示技术重心变化，不表示旧方法被完全替代。

02 · HISTORY

推荐算法发展历程综述

推荐模型的演进不是“模型越来越深”这么简单，而是表示、监督信号和服务约束不断共同变化。下面每一阶段都回答四个问题：压缩了什么、召回怎样做、排序怎样做、又留下了什么问题。

1990s—2013

邻域与低秩表示

GroupLens 从相似用户借信号，ItemCF 把关系转到更稳定的物品侧；Netflix Prize 时代的 MF 用低维 user/item 坐标重构评分；FM 再把低秩交互推广到任意稀疏 field。召回与排序尚未完全分工，冷启动和时序仍弱。

2013—2017

深度级联定型

DSSM 的双塔结构让 item 向量可预计算，ANN 支撑亿级全库检索；GBDT+LR、Wide&Deep 和 DeepFM 在较小候选集恢复非线性交互。召回追求可索引，排序追求表达力，现代漏斗由此成形。

2017—2023

兴趣序列、Transformer 与多目标

MIND 用多个向量覆盖多意图；DIN/DIEN 建模候选相关兴趣；SASRec 与 BERT4Rec 把自注意力引入行为序列；MMoE 和 PLE 管理多个业务目标。模型开始从“用户是谁”转向“用户此刻处于什么状态”。

2023—2026

Semantic ID 与生成统一

TIGER 把全库召回转成受约束 Semantic ID 生成；OpenOneRec 探索列表生成和偏好对齐；HSTU 把长期行为流作为统一序列。新的难点变为合法解码、目录更新、吞吐、规模收益和全链路 ROI。

04 · GENERATIVE RECOMMENDATION

生成式召回、排序与召排融合

数学与任务导读发展与结果总结
RETRIEVAL

生成式召回

TIGER 先用 RQ-VAE 将 item 压成多级 Semantic ID，再由序列模型自回归生成 ID。语义相近 item 共享前缀，利于冷启动；必须约束到合法 ID trie 并处理码本漂移。

RANKING

生成式排序

把候选与上下文放入语言模型，让模型输出排列或相关性 token。优势是指令与语义迁移；瓶颈是候选长度、位置偏差、校准、吞吐和可复现实验。

UNIFICATION

召排融合

OneRec 直接生成推荐 session/list，并以奖励模型、DPO 对齐；HSTU 用面向高基数、非平稳行为流的序列转换单元统一预训练与下游任务。
