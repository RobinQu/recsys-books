# RecSys Atlas 教程编写规范

本文档汇总本项目在多轮评审中形成的额外要求，也是后续页面、Notebook 与测试的共同验收标准。

## 1. 目标读者与教学目标

- 默认读者具备基础代数与函数知识，不假设已经学过线性代数、概率论、微积分或深度学习。内容应保持易懂，但不在最终产物中使用教育阶段标签描述读者。
- 先解释问题、直觉和符号，再展示公式；公式必须说明每个符号、张量形状、计算步骤和工程含义。
- 抽象数学应配可手算的小例子、Python 数值验证和图表。通用知识集中在 3.0 数学基础，算法章节只复习本节需要的部分。
- 教程既要帮助初学者理解，也要让工程人员看到训练、离线推理、在线服务、指标和工业约束的完整闭环。

## 2. 总体章节安排

1. 召回与排序双视角的完整技术演进：解释全库检索与候选精排的不同约束，以及级联系统、序列建模和生成式统一的演进。
2. 推荐算法发展历程综述：按邻域/低秩、深度级联、序列与多任务、生成式推荐四个阶段梳理关键变化。
3. 主流模型讲解与实战：
   - 3.0 数学基础：行为矩阵、向量、点积、余弦、矩阵乘法、概率、Sigmoid、LogLoss、梯度下降与常用指标。
   - 3.1 经典算法：协同过滤、MF、FM、GBDT+LR。
   - 3.2 召回模型：DSSM 双塔、MIND。
   - 3.3 排序模型：DeepFM、DIN、DIEN。
   - 3.4 多目标模型：MMoE、PLE。
   - 3.5 Transformer 序列推荐：SASRec 实战，以及 GRU4Rec、BERT4Rec、HSTU 的演进与边界。
4. 生成式推荐：4.1 生成式召回/排序/召排融合与工业实践；4.2 OpenOneRec；4.3 DLRM HSTU。
5. 论文与官方资料证据清单。

## 3. 每个大章节的固定结构

1. **开篇导读与数学基础 Notebook**：每个大章都独立介绍内容布局、横向差异、应用场景、来源论文和本章共享数学，并配简单 Python 图解。
2. **独立算法章节**：一个算法一个 Notebook，不把多个算法塞进同一个长 Notebook。DIN 与 DIEN、MMoE 与 PLE 也分别讲解。
3. **章节总结**：读取各算法 Notebook 实际写出的结果文件，统一指标口径并讨论选型；禁止手填演示指标。

## 4. 每个算法 Notebook 的固定结构

1. Goal：学完能回答的问题、先修知识和本节边界。
2. Paper & Context：原始论文、研究问题、关键假设、相对前代方法的变化，以及论文结论不能外推的部分。
3. Math by Hand：直觉化公式阅读、符号表、张量形状、可手算例子、Python 数值演示和至少一幅原理图。
4. Data：真实开源数据集背景、字段、任务派生规则、时间切分、正负样本和防泄漏检查；禁止随机制造交互、标签或行为序列。
5. Model & Framework：模型结构、工业框架类、smoke/full 两档、关键超参数和生产映射。
6. Train：可重复 seed、训练循环、损失曲线和必要的合理性检查。
7. Inference：展示召回索引/候选打分/序列解码等真实推理路径，不以训练前向代替线上推理。
8. Test & Metrics：使用与任务匹配的离线指标，并至少给出一个基线或消融对照。
9. Results Discussion：引用已执行输出中的具体数字，解释结果、失败案例、数据局限、成本和适用边界。
10. Checks & Next Steps：自动断言、复现实验说明、扩展到完整数据与生产服务的步骤。

## 5. 实现与数据要求

- 3.1 使用 NumPy、PyTorch、scikit-learn、XGBoost 等基础框架，展示关键计算而非隐藏在高级 API 中。
- 3.2 及以后使用 Torch-RecHub 的对应工业模型类执行 smoke 训练；同时给出 TorchEasyRec 或官方仓库的 full-profile 配置映射。
- 每本 Notebook 默认读取仓库内固定版本的 GroupLens MovieLens latest-small；允许确定性截取和任务派生，但所有原始行为、评分和时间戳必须来自真实数据，并公开派生标签规则。
- 完整数据默认按时间切分，训练只看过去；报告数据规模、稀疏度、标签比例和序列长度。
- 训练结果写入 `results/<chapter>/*.json`，章节总结只读取这些产物。

## 6. Web、交互与验收要求

- HTML 是 Notebook 的阅读友好版本，通过 nbconvert 生成；首页不嵌入 Notebook，直接链接到大章数学导读、独立算法和结果总结。
- 不设置额外章节详情层级；旧 `/chapters/*` 地址只做兼容跳转到对应数学导读，信息迁移到导读 Notebook 和首页算法卡片。
- 每个 Notebook 有站点风格一致的容器页、返回导航、上一篇/下一篇和进入 JupyterLab 的入口。
- Jupyter 使用项目专用端口 `8889` 与固定 token，入口必须能直接打开目标文件。
- 提供 Dockerfile、docker-compose 与自动化测试；`docker compose run --rm test` 必须执行测试、全部 Notebook 和全部 HTML 预览。
- 自动测试至少验证：章节拆分、必需叙事段落、实际框架类、图表、结果产物、无错误输出、导航链接和 Jupyter URL。
